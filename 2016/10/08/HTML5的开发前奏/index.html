HTML5的开发前奏

本小节知识点

  1. iOS移动端学习HTML5的原因
  2. HTML5的基础知识
  3. Web开发新时代

1.iOS移动端学习HTML5的原因

  • 1.接下来要学习的语言是HTML语言,HTML语言用在很多电商类型的项目上
  • 2.BAT这类大型的公司,对项目的框架进行改革,传统的是前端,后端,移动端;BAT这类大型的公司现在整合前端和移动端(大前端)
  • 3.这对我们提出了更高的要求,需要学习HTML5的原因是跨平台的项目

2.HTML5基础知识

  • 1.什么是HTML5?

    • HTML5分为两个部分?HTML和5两个部分,HTML是网页,5是第五代

  • 2.移动端的为啥要学习HTML5呢?

    • 得益于两大背景:(1)2014年才有W3C制定完HTML5的标准,耗时8年(一流的公司制定标准,二流的公司执行标准),在软件开发行业,标准最重要,各个地方的开发者就会开发各种框架,维护各种资源(2)移动先行,在以前做出来的网页都是显示在PC端的,自从2014年标准制定出来之后,她优先考虑移动端,在ipad上和智能手机上能不能很好的显示出来,同时兼容PC端
    • 内在的原因是跨平台(利用HTML5开发出来的应用能运行在所有的浏览器的平台),HTML5的平台是浏览器,运行起来很方便

  • 3.HTML5的优缺点

    • 优点:跨平台的成本比较低,运行起来很方便
    • 缺点:但是HTML5不能直接完成一些特定的功能,比如:拍照(ImagePickerController),访问相册…,要想实现这些特定的功能,需要借助一些桥梁(比如:JS)才能实现特定功能

  • 4.如何使用HTML5?

    • (1)自己编写大量的HTML5代码
    • (2)使用现成的一些框架(下面这四个框架适用于移动端,一个好的,强大的框架能体现一门语言的优越性)
      • sencha-touch
      • phoneGap
      • jQuery mobile
      • Bootstrap

    • jQuery mobile的简介

      • jQuery mobile是一个偏重型的框架

      • 下图是jQuery mobile 这个框架的一些demo的截图

      • 这个框架适配的是移动端的(移动端的主要是屏幕的适配)

      • Pad大小的界面如下图(细心一些会发现界面的大小发生变化了):

      • 手机大小的界面如下图:

      • HTML5的一个跨平台是非常廉价的,运行在手机上也是没有问题的

      • 上面这个截图看上去很像OC中的抽屉效果,适配移动端也是没有问题的

      • 看一下jQuery Mobile框架中的一些组件(相当于OC中的控件),看一下和原生的控件有什么区别

      • 可以看到HTML5的组件和原生的组件已经很像了

  • 5.手机APP的开发模式

    • (1)原生(纯OC)(市场占比40%左右)
    • (2)纯HTML5 (市场占比大概20%)
    • (3)原生 + HTML5 (市场占比40%,占大多数)

  • 6.为什么要学习HTML5?

    • (1)目前的一种大趋势(大前端时代)
    • (2)增加面试,开发竞争力
      • android 程序员,java,服务器,HTML5(安卓会的,iOS也要会一些)

  • 7.IT公司的职位的划分

    • (1)平面设计师 : 作图,切图,HTML,CSS;如果需要@4X的图,找UI设计师
    • (2)前端工程师 : HTML,CSS,JavaScript,模板设计;嵌入html5的界面可以找前端
    • (3)后台工程师 : 服务器(Java,.Net,PHP),数据库(SQLite[用于嵌入式移动端的],MySQL[免费,小型公司用的比较多],Oracle(大公司用的比较多,像京东,途牛啊),淘宝没有用到数据库,数据库是关系型的,淘宝用的是云端(云数据)),注意:数据库的多表查询,增删改查,事物这些东西经常会被问到,我们和后端工程师之间的交流是和服务器进行交互
    • (4)移动工程师 :（iOS/android）手机UI界面（OC、HTML5）、跟服务器交互

  • 8.开发工具

    • 1> Android
      eclipse 、MyEclipse、android studio
    • 2> iOS
      Xcode

    • 3> HTML5
      eclipse、MyEclipse
      后端喜爱

    • Dreamwaver
      网⻚三剑客 Dreamwaver、Flash、Fireworks -> Adobe(Photoshop,JS)(UI设计师和传统的前端工程师)

    • WebStorm
      前端喜爱,⼤神编辑器,默认集成各种各样的插件,配⾊完美(专业的前端工程师)

3.Web开发新时代

  • 1.Web1.0阶段

    • 主流技术: Html + CSS(在一些古老的政府的网站上能看到)

  • 2.Web2.0阶段

    • 主流技术: Ajax(JavaScript/DOM/异步数据请求)(好看的界面大多数是通过Ajax出来的,现在大部分网页还是基于Web2.0)

  • 3.Web3.0阶段

    • 主流技术: HTML5 + CSS3
    • HTML5的亮点: Canvas HTML5音视频 Web存储 Geolocation Workers多线程处理
      • 为什么说HTML5的技术是革命性的?
      • 首先是Canvas,通过Canvas实现一种很炫的灯光秀,还有现在玩的一些HTML5小游戏,一些经典的特效,3D建模等都是Canvas实现的,想往游戏方向发展的话,可以学这个Canvas
      • HTML5音视频的功能实现要比OC简单,而且所有的功能都有
      • Web存储,之前html5和原生应用之间的交互不是很好,Web3.0出来之后,可以对数据进行缓存,网络不好的时候,可以加载缓存里面的,缓存用的是SQLite
      • Geolocation,比原生的应用的定位更加强大,做一些所罗门类型的应用(社交类的定位)
      • Workers多线程处理,对任务的处理是多线程的,性能更好
    • CSS3的亮点:设计动画,2D变形,N多的新特性
